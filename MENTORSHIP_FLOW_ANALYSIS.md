## üóÑÔ∏è **1. DATABASE LAYER**

### **Core Tables Structure:**

| Table | Purpose | Key Relationships |
|-------|---------|------------------|
| `mentorship_templates` | Stores template definitions (e.g., "Baseline Entrepreneur Info") | ‚Üí `mentorship_categories` |
| `mentorship_categories` | Groups questions into sections (e.g., "Business Info", "SWOT Analysis") | ‚Üí `mentorship_questions` |
| `mentorship_questions` | Individual form questions with validation rules | ‚Üí `mentorship_responses` |
| `mentorship_sessions` | Session instances linking companies to templates | ‚Üí `mentorship_responses` |
| `mentorship_responses` | User answers stored as JSON | Links sessions + questions |
| `mentorship_tasks` | Generated action items | ‚Üí `mentorship_sessions` |
| `mentorship_task_triggers` | Rules for automatic task creation | ‚Üí `mentorship_questions` |
| `mentorship_triggered_tasks` | Log of triggered tasks | ‚Üí `mentorship_sessions` |

### **Key Database Relationships:**
```sql
mentorship_templates (1) ‚Üí (many) mentorship_categories
mentorship_categories (1) ‚Üí (many) mentorship_questions  
companies (1) ‚Üí (many) mentorship_sessions
mentorship_templates (1) ‚Üí (many) mentorship_sessions
mentorship_sessions (1) ‚Üí (many) mentorship_responses
mentorship_questions (1) ‚Üí (many) mentorship_responses
```

### **API Request/Response Flow:**

**Example: Getting Template with Questions**
```
GET /api/mentorship/templates.php?id=1
‚Üí templates.php calls MentorshipTemplate::getWithDetails(1)
‚Üí MentorshipTemplate joins categories and questions tables
‚Üí Returns JSON: { success: true, data: { template + categories + questions } }
```

**Example: Creating Session**
```
POST /api/mentorship/sessions.php
Body: { company_id: 1, template_id: 1, session_date: "2025-01-15" }
‚Üí sessions.php calls MentorshipSession::create($data)
‚Üí Creates record in mentorship_sessions table
‚Üí Returns JSON: { success: true, data: session, id: 123 }
```

### **Core TypeScript Interfaces:**

**From `src/models/mentorship.ts`:**

interface IMentorshipTemplate {
  id?: number;
  name: string;
  description: string;
  program_type: string;  // 'baseline', 'financial_turnaround'
  version: string;

interface IMentorshipCategory {
  id?: number;
  template_id: number;
  name: string;           // "Entrepreneur Info", "SWOT Analysis"

interface IMentorshipQuestion {
  id?: number;
  category_id: number;
  key: string;           // "business_name", "sales_ability"
  label: string;         // "Business Name", "Sales Ability (1-10)"

  required: boolean;
  placeholder?: string;
  validation?: any;
  options?: string[];    // For dropdown/multi-select
  items?: string[];      // For list type

interface IMentorshipSession {
  id?: number;

  template_id: number;
  session_date: string;
  status: 'scheduled' | 'in_progress' | 'completed' | 'reviewed';

  notes?: string;
  responses?: IMentorshipResponse[];
}

interface IMentorshipResponse {
  id?: number;
  session_id: number;
  question_id: number;
  question_key: string;
  response_value: any;    // JSON storage for any response type
  response_text?: string;
}

## üîÑ **7. COMPLETE DATA FLOW EXAMPLE**

### **Excel to Digital Form Execution:**

**Step 1: Template Creation (Admin)**
```
Admin creates "Baseline ‚Äì Entrepreneur Information" template:
1. Component: template-manager.component.ts
2. Service: mentorshipService.createTemplate()
3. API: POST /api/mentorship/templates.php
4. PHP: MentorshipTemplate::create()
5. Database: INSERT into mentorship_templates, mentorship_categories, mentorship_questions
```

**Step 2: Session Execution (Mentorship)**
```
Mentor executes session with company:
1. Component: mentorship-session-manager.component.ts
2. Service: mentorshipService.searchCompanies() ‚Üí mentorshipService.createSession()
3. API: GET /api/companies/list.php ‚Üí POST /api/mentorship/sessions.php  
4. PHP: Company::search() ‚Üí MentorshipSession::create()
5. Database: SELECT from companies ‚Üí INSERT into mentorship_sessions
```

**Step 3: Response Collection**
```
User fills form and saves responses:
1. Component: Dynamic form built from template structure
2. Service: mentorshipService.saveSessionResponses()
3. API: POST /api/mentorship/sessions.php?action=save_responses
4. PHP: MentorshipResponse::saveMultiple()
5. Database: INSERT/UPDATE mentorship_responses with JSON values
```

**Step 4: Task Generation (Automatic)**
```
System triggers tasks based on responses:
1. PHP: MentorshipResponse triggers check mentorship_task_triggers
2. Database: INSERT into mentorship_triggered_tasks based on trigger conditions
3. Example: If "VAT Compliance" = "Non-Compliant" ‚Üí Create "VAT Assistance" task
```
